# Brainstorming Report: 发财加速器 (Fortune Accelerator)

## 1. Topic Candidates (候选主题)
- 乐透号码中奖趋势预测 (当前选定)
- 历史中奖数据可视化分析
- 基于概率论的随机数生成器 quality 评估

## 2. Selected Topic (选定主题)
- **发财加速器**：利用深度学习（Bi-LSTM + Attention）对乐透历史数据进行时序建模，辅助个人决策。

## 3. Why this Topic? (为什么选这个)
- 个人兴趣驱动，希望通过技术手段量化随机性。
- 探索 AI 模型在极端随机、低频数据下的表现。

## 4. Problem to Solve (要解决的问题)
- 解决在购买乐透时盲目选号的问题，提供基于历史数据的参考建议。
- 建立一个自动化的回测系统，验证预测模型的实际命中率。

## 5. User & Use Case (用户与场景)
- **用户**：对概率论有兴趣的个体投资者（如开发者本人）。
- **场景**：在每期乐透开奖前，运行脚本获取预测号码。

## 6. Data Ideas (数据构思)
- **实时数据**：通过东航乐透官方 API 获取最新数据。
- **模拟数据**：生成纯随机序列用于模型的压力测试和过拟合检测。

## 7. Key Features (核心功能)
- 自动数据同步引擎。
- 基于 Attention 机制的 Bi-LSTM 预测模型。
- 历史回测验证系统（带可视化展示）。

## 8. Technical Challenges (技术挑战)
- 如何在高度随机的数据中提取微弱的模式。
- 防止模型对历史数据的简单记忆（过拟合）。

## 9. Out of Scope (非必要功能)
- **全自动投注系统**（明确不做，涉及安全和法律风险）。
- 手机 App 开发（保持为 Python-first 的实验项目）。

## 10. Success Criteria (成功标准)
- 模型在 20 期回测中的命中率高于纯随机选号。
- 系统能够一键运行并生成清晰的推荐列表。

## 11. Ethical Considerations (伦理考虑)
- 明确项目仅供学习娱乐，不构成投资建议。
- 遵守相关数据获取的频率限制。

## 12. Next Steps (后续计划)
- 细化模型架构，调整 Attention 层权重。
- 完善回测报告的视觉呈现。
